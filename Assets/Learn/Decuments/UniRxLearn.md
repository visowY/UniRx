# <center>UniRx</center>


## 使用UniRx的原因：

* 1. 对于协程的改进
    * 协程不能有返回值
    * 协程不能处理异常
* 2. 多线程变成更加容易
* 3. 对于UI事件的包装-> 转换成UniRx流

## Introduction 

* Reactive Programming : Reactive programming is programming with asynchronoud datas streams;(响应式变成是使用异步的数据流编程)

* stream : A stream is a sequence of ongoing events ordered in time.
